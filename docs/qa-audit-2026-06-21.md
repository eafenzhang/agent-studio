## 一、首页（HomePage）

### 1. 发送请求未携带用户选择的模型 / 模式 / 专家 / 工具
- **位置**：`src/pages/HomePage.tsx:206-230`（`handleSend`）
- **复现步骤**：
  1. 在首页选择任意模型、模式、专家或工具。
  2. 在输入框输入内容并点击发送。
  3. 观察网络请求 `POST /api/conversations/:id/messages` 的请求体。
- **期望行为**：请求体应包含 `model`、`mode`、`expert_id`、`tools` 等参数，后端据此路由到正确的模型与专家。
- **实际行为**：请求体只包含 `{ content }`，用户选择的模型、模式、专家、工具全部被忽略。
- **优先级**：P1
- **根因分析**：`handleSend` 直接调用 `api.sendMessage(convId, { content })`，未把 `selectedModel / selectedMode / selectedExpert / selectedTools` 拼入 payload。
- **处理建议**：需工程师修复，在 `sendMessage` 时补充上述字段；同步更新 `SendMessagePayload` 类型与后端契约。
