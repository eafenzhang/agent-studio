## 三、P0 需求功能规格

### P0-1 消息操作栏

**User Story**
- As a 普通用户, I want 对 AI 回复进行复制/重新生成/点赞点踩 so that 我能快速复用内容并反馈质量
- As a 高级用户, I want 编辑已发送的消息并重新提交 so that 我能修正输入错误后获得更好的回复

**验收标准**
- [ ] 每条 AI 消息 hover 时显示操作栏（复制 / 重新生成 / 👍 / 👎）
- [ ] 复制按钮点击后将消息纯文本复制到剪贴板，显示 toast "已复制"
- [ ] 重新生成触发重新发送上一条用户消息（保留原对话历史），新回复替换当前 AI 消息
- [ ] 点赞/点踩记录反馈（存储到后端 feedback endpoint 或本地 state）
- [ ] 用户消息 hover 显示编辑按钮，点击进入编辑模式（inline textarea），支持 Enter 重新提交
- [ ] 操作栏在移动端/窄屏下自动适配为图标模式

**UI 描述**
```
┌──────────────────────────────────────────┐
│  [AI Avatar]  AI 回复内容...              │
│                                          │
│  📋 复制  🔄 重新生成  👍  👎           │ ← hover 出现
└──────────────────────────────────────────┘
```
- 操作栏位于 `msg-bubble` 底部外侧，水平排列
- 按钮样式：ghost 风格，12px 字体，icon + 文字
- 点赞后按钮高亮为绿色，点踩后高亮为红色

**技术可行性**：✅ 纯前端可实现。复用现有 `copyToClipboard()` util，重新生成复用 `chatService.sendMessage()`。

---

### P0-2 文件上传/附件

**User Story**
- As a 用户, I want 在对话中拖拽或选择文件上传 so that AI 能读取文件内容并基于文件回答
- As a 用户, I want 看到已上传文件的缩略图/文件名 so that 我确认上传正确

**验收标准**
- [ ] 消息输入区域支持拖拽文件（drag & drop），显示虚线高亮区域
- [ ] 文件菜单的"上传文件/截屏/文件夹"入口变为可用
- [ ] 文件上传后显示附件标签（文件名 + 文件大小 + 删除按钮），位于输入框上方
- [ ] 支持多文件同时上传（最多 10 个，单文件 ≤50MB）
- [ ] 上传进度条显示（利用 fetch 的 ReadableStream 或后端返回进度）
- [ ] 图片文件显示缩略图预览

**UI 描述**
```
┌──────────────────────────────────────────┐
│  📎 report.pdf (2.3MB) [✕]              │ ← 附件标签行
│  🖼️ screenshot.png [✕]                  │
│  ┌──────────────────────────────────┐   │
│  │ 请输入消息...                    │   │
│  └──────────────────────────────────┘   │
│  [📎 附件] ...    [⚙️模式]  [发送]     │
└──────────────────────────────────────────┘

拖拽区域虚线边框高亮：
┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐
│        📤  拖拽文件到此处上传         │
└ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘
```

**技术可行性**：⚠️ 需后端配合。文件上传需后端提供 multipart upload endpoint（如 `POST /api/fs/upload`），前端实现拖拽区域 + File API + FormData 上传。

### P0-3 智能任务步骤可视化增强

**User Story**
- As a 用户, I want 看到 AI 正在执行的具体步骤及其耗时 so that 我了解任务进度并预估完成时间
- As a 用户, I want 点击展开步骤查看详细日志 so that 我能诊断任务失败原因

**验收标准**
- [ ] 任务进度面板默认展开前 3 步，后续步骤折叠
- [ ] 每个步骤显示：状态图标 + 步骤标题 + 耗时（如 `1.2s`）+ 展开箭头（若有详情）
- [ ] `in_progress` 步骤显示旋转动画指示器
- [ ] `completed` 步骤显示 ✅ + 耗时
- [ ] `error` 步骤显示 ❌ + 错误摘要，点击展开查看完整错误堆栈
- [ ] 折叠/展开面板增加 smooth 动画
- [ ] 步骤条显示总体进度条（如 `3/7 已完成`）

**UI 描述**
```
┌──────────────────────────────────────────┐
│  📋 任务进度 (3/7)          ▁▃▅▇█████░░ │ ← 进度条
│  ─────────────────────────────────────── │
│  ✅ 分析用户需求           0.8s          │
│  ✅ 查询知识库             2.1s    [▶]   │ ← 可展开查看日志
│  ⏳ 生成回复内容...       进行中          │ ← 旋转动画
│  ○ 格式检查               等待中          │
│  ○ 输出结果               等待中          │
│  ─────────────────────────────────────── │
│  显示更多 (点击展开)                      │ ← 折叠的步骤
└──────────────────────────────────────────┘
```

**技术可行性**：✅ 纯前端可实现。扩展 `task-progress.js` 组件，增加进度条、耗时、展开详情。后端已通过 WS `TASK_STEP_UPDATE` 事件推送步骤数据。

### P0-4 对话内 Artifacts 内联预览

**User Story**
- As a 用户, I want AI 生成的 HTML/代码/文档直接在对话中预览 so that 我不用切换到产物页查看
- As a 用户, I want 点击内联预览在新标签页打开全屏预览 so that 我能更详细地查看大型产物

**验收标准**
- [ ] AI 消息中如果包含产物链接，自动在消息下方渲染内联预览卡片
- [ ] HTML 类型产物：在 sandboxed iframe 中预览（350px 高度，可拖拽调整）
- [ ] 代码类型产物：语法高亮代码块，带"复制代码"按钮
- [ ] Markdown 类型产物：渲染后的预览
- [ ] 图片类型产物：自适应缩略图，点击放大
- [ ] 预览卡片右上角有"在新标签页打开"按钮 + "下载"按钮
- [ ] 卡片有圆角边框 + 轻微阴影，与消息气泡视觉区分

**UI 描述**
```
┌──────────────────────────────────────────────┐
│  [AI]  这是生成的网页：                       │
│  ┌────────────────────────────────────────┐  │
│  │  🌐 index.html                  [↗] [⬇]│  │
│  │  ┌──────────────────────────────────┐  │  │
│  │  │     (iframe sandbox preview)     │  │  │
│  │  │                                  │  │  │
│  │  └──────────────────────────────────┘  │  │
│  └────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

**技术可行性**：✅ 纯前端可实现。在 `chat.js` 的 `renderMessage` 中检测产物链接/内容，内联渲染预览卡片。复用 `artifacts.js` 的预览逻辑。

## 七、Open Questions
