# PRD：Agent Studio Desktop 对齐 WorkBuddy 功能

- **版本**：v1.0
- **日期**：2025-07-14
- **产品经理**：许清楚（Xu）
- **项目名称**：agent_studio_desktop
- **语言**：中文
- **技术栈**：Tauri 2.x + Vite + 纯 HTML/CSS/JS + AionCore（Rust，HTTP REST + WebSocket）

---

## 一、原始需求

将 **Agent Studio Desktop**（基于 Tauri 2.x + AionCore 的桌面 AI 工作台）现有的占位 / 未实现功能，对齐成熟产品 WorkBuddy 的核心功能水平，使其具备完整的 AI 对话体验、工具调用可观测性、专家系统、产物展示及技能市场能力。

---

## 二、产品目标

| # | 目标 | 度量标准 |
|---|------|----------|
| G1 | **完整对话体验**：用户能在 Agent Studio 中完成与 WorkBuddy 同等质量的多轮 AI 对话，包含流式渲染与工具调用可视化 | 流式消息延迟 < 200ms；工具调用结果可在折叠卡片内查看 |
| G2 | **功能完整度对齐**：将现有占位/未实现模块（专家、产物、项目、工具、任务进度）全部接入真实数据 | 所有侧边栏导航项均可正常使用，无"暂未开放"占位 |
| G3 | **AI 执行可观测性**：用户能清晰看到 AI 当前正在执行的步骤、工具调用详情及生成的产物 | 任务进度面板实时更新；Artifacts 支持 HTML 预览与代码高亮 |

---

## 三、用户故事

1. As a **开发者用户**，I want **点击侧边栏会话并进入完整对话视图、看到 AI 流式回复**，so that **我能进行连贯的多轮对话，而不是盯着空白页**。
2. As a **开发者用户**，I want **在对话中看到 AI 调用了哪些工具、传入了什么参数、得到什么结果**，so that **我能理解 AI 的推理过程，方便调试与审查**。
3. As a **开发者用户**，I want **在专家页浏览、启用系统内置专家或自定义专家**，so that **我能快速切换不同领域的 AI 助理角色**。
4. As a **开发者用户**，I want **在产物页查看 AI 生成的文件（HTML 预览、代码、文档）**，so that **我无需手动打开文件即可预览 AI 的输出结果**。
5. As a **开发者用户**，I want **在工具页安装/卸载 Skill 技能、管理 MCP 服务器**，so that **我能扩展 AI 的能力边界，按需启用工具**。

## 五、需求池

### P0 — 必须实现（核心体验断点）

| ID | 需求 | 说明 |
|----|------|------|
| P0-1 | **会话详情页** | 点击侧边栏会话卡片，打开完整对话视图（消息列表、输入区），支持多轮上下文 |
| P0-2 | **流式消息渲染** | 通过 WebSocket 接收 AI token 流，逐字渲染到消息气泡；光标闪烁动画 |
| P0-3 | **工具调用展示** | AI 调用工具时，在消息流中插入折叠卡片：工具名称 / 入参（JSON 折叠）/ 执行结果 |
| P0-4 | **任务进度面板** | 多步任务执行时，在对话上方展示步骤列表，实时更新 in_progress / completed 状态 |
| P0-5 | **专家页真实数据接入** | 调用 GET /api/assistants（或专家专属接口），渲染专家卡片（头像/名称/描述/标签），支持启用/切换 |

### P1 — 应该实现（功能完整性）

| ID | 需求 | 说明 |
|----|------|------|
| P1-1 | **产物页真实数据接入** | 调用 Artifacts API，展示 AI 生成的文件；HTML 类型支持 iframe 预览，代码类型支持语法高亮 |
| P1-2 | **技能市场（Skills）** | 调用 Skill 相关 API，展示已安装技能列表，支持启用/禁用/卸载；预留安装入口 |
| P1-3 | **聊天工具栏联动** | 模型选择下拉接入 Provider 列表；专家选择接入专家列表；工具开关联动 MCP/内置工具状态 |
| P1-4 | **记忆功能接入** | 调用记忆 API，在设置页展示真实记忆条目，支持查看/删除单条记忆 |
| P1-5 | **侧边栏会话分组** | 会话列表按今天/昨天/更早分组，支持会话重命名、删除 |

## 六、UI 设计稿（关键交互描述）

### 6.1 会话详情页（P0-1）

```
┌─────────────────────────────────────────────────────┐
│ 侧边栏           │  对话视图                         │
│                  │  ┌─────────────────────────────┐  │
│ [今天]           │  │ 消息气泡（用户，右对齐）      │  │
│  ○ 会话 A  ●    │  │ 消息气泡（AI，左对齐）       │  │
│  ○ 会话 B        │  │  └ 工具调用折叠卡（P0-3）    │  │
│ [昨天]           │  │ [任务进度面板（P0-4）]       │  │
│  ○ 会话 C        │  └─────────────────────────────┘  │
│                  │  ┌─────────────────────────────┐  │
│                  │  │ 输入区：文本框 + 发送按钮     │  │
│                  │  │ 工具栏：模型▼ 专家▼ 工具▼   │  │
│                  │  └─────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

- 点击侧边栏会话卡片 → 右侧加载对话历史，滚动到最新消息
- AI 回复中：消息气泡末尾显示光标 `▋`，token 逐字追加
- 工具调用：消息气泡下方插入折叠卡，默认折叠，点击展开查看入参/结果

### 6.2 工具调用折叠卡（P0-3）

```
▶ 🔧 read_file  [展开]
  ├ 入参：{ "path": "/src/main.rs" }
  └ 结果：[文件内容，超长时截断 + "查看全文"]
```

- 执行中：显示旋转 spinner + "正在执行…"
- 完成：绿色对勾；失败：红色叉号 + 错误信息

### 6.3 任务进度面板（P0-4）

```
┌──── 任务执行中 ────────────────────────────┐
│ ✅ 分析需求                                 │
│ ⏳ 搜索相关代码  ←── 当前步骤（spinner）   │
│ ○  生成实现方案                             │
│ ○  写入文件                                 │
└─────────────────────────────────────────────┘
```

位置：对话视图顶部，任务结束后自动收起（可手动点击折叠）。

### 6.4 专家页卡片（P0-5）

```
┌─────────────────┐  ┌─────────────────┐
│  [头像]          │  │  [头像]          │
│  专家名称        │  │  专家名称        │
│  角色描述（2行） │  │  角色描述（2行） │
│  [标签] [标签]   │  │  [标签] [标签]   │
│           [启用] │  │         [已启用] │
└─────────────────┘  └─────────────────┘
```

- 网格布局（3列），支持搜索过滤
- 启用时：卡片边框高亮，按钮变为"已启用"（可再点切换）

### 6.5 产物页（P1-1）

```
┌─── 产物列表（左） ───┬─── 预览区（右）────────────────┐
│ 📄 index.html        │  [iframe 沙盒预览 / 代码高亮]   │
│ 📝 report.md         │                                  │
│ 💻 solution.py       │                                  │
└──────────────────────┴──────────────────────────────────┘
```
